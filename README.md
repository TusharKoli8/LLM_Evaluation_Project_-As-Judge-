# LLM As Judge

## Overview

LLM As Judge is a project that demonstrates the use of Large Language Models (LLMs) to evaluate the quality of answers generated by other LLMs. The system uses one LLM to generate responses to questions and another LLM to act as a "judge" that evaluates those responses against a set of golden standard answers.

This project is particularly useful for understanding how LLMs can be leveraged for automated evaluation in natural language processing tasks, such as question-answering systems.

## How It Works

1. **Dataset Loading**: The project loads a golden dataset containing questions and their corresponding correct answers (stored in `data/goldendataset.json`).

2. **Answer Generation**: For each question in the dataset, an LLM (configured as the "generator") produces an answer.

3. **Evaluation**: Another LLM (configured as the "judge") evaluates the generated answer by comparing it to the golden answer. The evaluation covers criteria such as correctness, relevance, completeness, faithfulness, hallucination, and reasoning quality.

4. **Results Storage**: All results, including questions, golden answers, generated answers, and evaluations, are saved to a CSV file (`data/evaluation_results.csv`).

## Features

- Automated evaluation of LLM-generated answers using another LLM as a judge.
- Support for custom datasets (JSON format).
- Configurable LLMs via environment variables.
- Easy-to-run Python scripts for dataset generation and evaluation.
- Results exported to CSV for further analysis.

## Installation

1. Clone or download the project repository.

2. Install the required dependencies:
   ```
   pip install -r requirements.txt
   ```

3. Set up your environment variables:
   - Create a `.env` file in the project root.
   - Add your Groq API key: `GROQ_API_KEY=your_api_key_here`
   - Specify the models: `MODEL1=your_generator_model` and `MODEL2=your_judge_model` (e.g., `MODEL1=gemma-7b-it`, `MODEL2=llama3-8b-8192`)

## Usage

1. **Generate the Golden Dataset** (if not already present):
   ```
   python src/generate_dataset.py
   ```
   This creates `data/goldendataset.json` with sample questions and answers about databases.

2. **Run the Evaluation**:
   ```
   python src/main.py
   ```
   This will process each question, generate answers, evaluate them, and save results to `data/evaluation_results.csv`.

3. **View Results**: Open `data/evaluation_results.csv` to see the evaluation results.

## Configuration

The project uses the Groq API for LLM interactions. Ensure you have a valid API key and configure the models in your `.env` file.

- `GROQ_API_KEY`: Your Groq API key
- `MODEL1`: Model for answer generation (e.g., `gemma-7b-it`)
- `MODEL2`: Model for evaluation (e.g., `llama3-8b-8192`)
